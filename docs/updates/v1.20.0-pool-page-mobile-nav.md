### Improved Disconnected UX

**Old Pattern (Removed):**
- Full-page blocker when wallet not connected
- User couldn't see any interface
- High friction for new visitors

**New Pattern:**
- Page content always visible
- Cards show contextual empty states:
  - "Connect wallet to view position"
  - "No fees to claim"
  - "Add liquidity to earn fees"
  - "No position to remove"
- Action buttons transform to "Connect Wallet"

### Always-Visible Cards

All 4 Pool page cards are now always rendered with appropriate empty states:

1. **Your Position**
   - Has position: Shows DGKO + KLV amounts, pool share %
   - No position: "No liquidity position yet"
   - Not connected: "Connect wallet to view position"

2. **Claim Fees**
   - Has fees: Shows pending amounts + Claim button
   - No fees: "No fees to claim" + "Add liquidity to earn fees"
   - Not connected: "Connect wallet to view fees"

3. **Add Liquidity**
   - Connected: Normal input form + "Add Liquidity" button
   - Not connected: Form visible but button shows "Connect Wallet"

4. **Remove Liquidity**
   - Has position: Slider + percentage buttons
   - No position: "No position to remove"
   - Not connected: "Connect wallet to remove"

## ğŸ¨ Design Patterns Established

### 1. Progressive Disclosure
Never block entire pages with wallet connection overlays. Let users explore first.

### 2. Always-Visible Cards
Cards remain visible with contextual empty states rather than hiding completely.

### 3. Connect Wallet Button Pattern
Action buttons transform to "Connect Wallet" when not connected, rather than blocking the entire action.

These patterns are now documented in:
- `/admin/design-system` â†’ UX Patterns section
- `docs/DESIGN_PHILOSOPHY.md` â†’ Principle #5
